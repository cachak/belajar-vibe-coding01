Isi dari planning adalah sebagai berikut:

buat api untuk get profile user yang sedang login
endpoint: 
- GET /api/v1/auth/me
- GET /api/v1/users/me

headers : 
- Authentication : Bearer <token> (token adalah token yang ada di tabel sessions)

description : 
- endpoint memang ada dua yaitu /api/v1/auth/me dan /api/v1/users/me, akan tetapi ini menerpakan validasi yang sama, dengan service getUserById nya yang dipanggil sama yang ada di module users
- validasi di lakukan di module auth atau buatkan middleware untuk validasi header token yang nanti bisa di gunakan di endpoint lain
- validasi apakan token sudah expire atau tidak, jika sudah expire maka kembalikan error
- gunakan service yang sudah ada untuk mengambil user atau buat baru jika memang tidak ada, karena ini ngambil usernya berdasarkan user_id

response body (success):
{
    "status": "ok",
    "message": "Get user profile successfully",

Dependency flow nya :
controller -> service -> repository -> entity/db
Artinya:
controller boleh panggil service
service boleh panggil repository
repository boleh import entity
controller jangan import repository langsung
service jangan berisi definisi table
entity jangan tahu tentang Elysia
